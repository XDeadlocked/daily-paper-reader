## 摘要
近期的文本到场景生成方法极大地减少了创建三维场景所需的人工工作。然而，这些方法通常仅关注于生成场景布局或生成对象，鲜有能同时生成两者的模型。即使借助大型语言模型（LLM），生成的场景布局通常仍较为简单。此外，生成的场景往往与输入文本中对物体形状、外观和空间布局的复杂描述不一致。我们提出了一种新的顺序式文本到场景生成范式，并引入了一种用于交互式场景创建的新型生成模型。其核心是一个三维自回归扩散模型 3D-ARD+，该模型将多模态标记序列上的自回归生成与下一对象三维潜变量的扩散生成相结合。为生成下一个对象，模型首先在场景空间中进行一次自回归步骤，生成粗粒度的三维潜变量，该步骤同时以当前的文本指令和已生成场景为条件。接着，模型进行第二步，在较小的对象空间中生成三维潜变量，这些潜变量可解码为细粒度的对象几何与外观。我们整理了一个包含 23 万个室内场景及其配对文本指令的大规模数据集用于训练。我们在具有挑战性的场景上评估了 70 亿参数的 3D-ARD+ 模型，展示了该模型能够根据文本指令中指定的复杂空间布局与语义生成并放置对象。

## Abstract
Recent text-to-scene generation approaches largely reduced the manual efforts required to create 3D scenes. However, their focus is either to generate a scene layout or to generate objects, and few generate both. The generated scene layout is often simple even with LLM's help. Moreover, the generated scene is often inconsistent with the text input that contains non-trivial descriptions of the shape, appearance, and spatial arrangement of the objects. We present a new paradigm of sequential text-to-scene generation and propose a novel generative model for interactive scene creation. At the core is a 3D Autoregressive Diffusion model 3D-ARD+, which unifies the autoregressive generation over a multimodal token sequence and diffusion generation of next-object 3D latents. To generate the next object, the model uses one autoregressive step to generate the coarse-grained 3D latents in the scene space, conditioned on both the current seen text instructions and already synthesized 3D scene. It then uses a second step to generate the 3D latents in the smaller object space, which can be decoded into fine-grained object geometry and appearance. We curate a large dataset of 230K indoor scenes with paired text instructions for training. We evaluate 7B 3D-ARD+, on challenging scenes, and showcase the model can generate and place objects following non-trivial spatial layout and semantics prescribed by the text instructions.

## 一、研究动机与核心问题

- **背景**  
  当前的文本到三维场景（Text-to-3D Scene）生成技术虽然能从文本自动创建场景，但往往面临“只能生成布局或对象”的局限。多数模型生成  
  - 简单的场景布局（如仅有几个家具的大致位置）；  
  - 或孤立的三维对象，而非语义一致的整体场景。

- **研究动机**  
  作者旨在建立一个**同时生成布局与对象形状的统一模型**，能在用户逐步输入的自然语言指令下，按照复杂空间关系依次生成完整3D场景。  

- **目标**  
  设计一种“**顺序式文本到场景生成**”范式，使模型能  
  1. 在多轮文本指令下逐步添加对象；  
  2. 保留之前生成的空间状态；  
  3. 在每一步理解文本语义并生成对应对象的几何与外观。

## 二、方法论：3D-ARD 与 3D-ARD+

### 1. 核心思想

提出 **3D Autoregressive Diffusion (3D-ARD+) 模型**，结合：
- **自回归机制**：控制多模态序列生成，使每一步生成依赖过往文本与已生成物体；
- **扩散生成**：在潜空间进行噪声-去噪过程，生成下一对象的三维潜变量。  

模型通过两阶段生成过程实现“布局—形状”联合生成：
1. **粗粒度场景空间生成（3D-ARD）**：在全局场景坐标中预测新对象的位置与体积占据。  
2. **细粒度对象空间生成（3D-ARD+阶段）**：针对当前对象局部区域进一步生成高分辨率几何与外观细节。

### 2. 技术细节与结构

- **输入形式**：
  - 文本指令经 BAGEL tokenizer 转换为序列化文本 token；
  - 已有物体的三维占据体积通过 3D-VAE 编码为理解 token；
  - 待生成对象的潜变量随机初始化为噪声后，作为生成 token。

- **多模态 Transformer 架构**：  
  采用 28 层 Transformer（7B 参数），带**广义因果注意机制**：
  - 文本 token 对先前文本与三维理解 token 采用因果注意；
  - 三维理解 token 对自身采用全局注意；
  - 生成 token 对文本与理解 token 使用因果注意，对自身全局注意。  

- **训练目标**：  
  使用“Rectified Flow”流匹配（Liu et al., 2023）目标，通过最小化  
  \[
  L(\theta) = E_{s,x_0,ϵ}\|v_{\theta}(x,s) - (ϵ - x_0)\|^2
  \]
  学习去噪向量场 \( v_\theta \)。

- **生成过程**：
  1. 每步从噪声采样，逐步去噪得到 3D 潜变量；
  2. 用 TRELLIS 3D-VAE 解码为占据体积；
  3. 由 TRELLIS 3DGS（高斯点云解码器）重建纹理与外观。

## 八、不足与局限

- **空间冲突**：  
  模型仍偶有物体重叠或间距异常，说明空间编码粒度不足。
- **基础对象规模问题**：  
  初始大型家具若过大，会影响后续生成空间分配。
- **计算成本高**：  
  纯Transformer结构导致推理时间为传统网络约5–10倍。
- **数据域局限**：  
  目前仅涵盖室内场景，缺乏户外或动态环境测试。
- **依赖VLM生成数据**：  
  指令质量受上游模型影响，可能引入描述偏差。
