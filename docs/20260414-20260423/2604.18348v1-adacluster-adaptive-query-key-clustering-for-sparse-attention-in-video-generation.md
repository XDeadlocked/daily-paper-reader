## 摘要
视频扩散 Transformer（DiT）由于注意力机制的二次复杂度而面临推理延迟过高的问题。现有稀疏注意力方法要么忽视语义相似性，要么无法适应跨层的异质 token 分布，从而导致模型性能下降。我们提出了 AdaCluster，这是一种无需训练的自适应聚类框架，可以在保持精度的同时加速 DiT 的生成。AdaCluster 对查询向量采用保持角度相似度的聚类方法以实现更高的压缩率，并为键设计了保持欧氏相似度的聚类方法，涵盖聚类数分配、基于阈值的自适应聚类以及高效关键簇选择。在单块 A40 GPU 上对 CogVideoX-2B、HunyuanVideo 和 Wan-2.1 的实验表明，AdaCluster 可在几乎无质量损失的情况下实现 1.67 至 4.31 倍的加速。

## Abstract
Video diffusion transformers (DiTs) suffer from prohibitive inference latency due to quadratic attention complexity. Existing sparse attention methods either overlook semantic similarity or fail to adapt to heterogeneous token distributions across layers, leading to model performance degradation. We propose AdaCluster, a training-free adaptive clustering framework that accelerates the generation of DiTs while preserving accuracy. AdaCluster applies an angle-similarity-preserving clustering method to query vectors for higher compression, and designs a euclidean-similarity-preserving clustering method for keys, covering cluster number assignment, threshold-wise adaptive clustering, and efficient critical cluster selection. Experiments on CogVideoX-2B, HunyuanVideo, and Wan-2.1 on one A40 GPU demonstrate up to 1.67-4.31x speedup with negligible quality degradation.

## 一、研究背景与核心问题

- **背景问题**：  
  视频扩散 Transformer（Video DiT）在高分辨率或长序列视频生成中表现出卓越的质量和可扩展性，但其注意力机制的计算复杂度随输入序列长度平方增长，导致推理时间极为庞大。  
  以 CogVideoX-2B、HunyuanVideo、Wan 系列为例，生成一个 720p、81 帧的视频在单张 A40 GPU 上需约 1691 秒，注意力部分占总时间的 75% 以上。  

- **现有方法的不足**：  
  - 固定稀疏模式的注意力（如 MInference、SVG）缺乏动态性。  
  - 动态稀疏注意（如 Top-K Attention、SparseAttn）虽灵活，但忽略 token 的语义分布差异；  
  - 聚类型算法（SVG2）使用统一聚类策略，忽视了不同层 Query 与 Key 分布的异质性。  

- **核心问题**：  
  如何在无需重新训练模型的前提下，自适应地聚类 Query 与 Key 向量以减少注意力计算，同时保持生成质量。

## 二、方法论：AdaCluster 框架设计与实现

核心思想为 **“自适应分层聚类 + 高效关键簇选择”**，通过为 Query 和 Key 分别设计针对性聚类策略，实现训练无关的稀疏注意力替代。

### 1. Query 聚类（角度相似性）
- **观察**：Query 向量长度不影响 Query-Key 打分的相对排序，故可进行标准化处理。  
- **聚类方式**：  
  将所有 Query 向量归一化至单位球面后，采用 **基于角度相似度（cosine / angle distance）** 的 K-means 聚类。  
- **效果**：使 Query 分布更紧凑，可获得高压缩比，实验表明未归一化情况下需 235 个簇，而归一化仅需 65 个簇，压缩比提升约 3.6×。  

### 2. Key 聚类（欧氏相似性 + 层级自适应）
- **挑战**：不同层的 Key 分布差异较大；需要保证聚类簇的紧致性以避免关键 token 丢失。  
- **方法结构**：
  - **自适应聚类数分配**：根据层内分布紧致度（Compactness Score = 1 / 平均重建误差 MSE）动态确定簇数量。  
  - **Multi-stage K-means**：逐层聚类 + 对离群样本再聚类，直到距离小于阈值 τ 或达到最大簇数上限 N_max。  
  - **跨时间步初始化复用**：扩散过程的连续步骤间 token 分布变化较小，故可在后续步骤中沿用上一步的聚类中心以节省计算。  

### 3. 关键簇高效选择（TensorQuest）
- **问题**：聚类后仍需判断哪些簇对应关键的 Key-Value 对。  
- **传统方法 Quest 的不足**：在 GPU 上计算低效，全部由 CUDA Core 执行。  
- **改进方案：TensorQuest**  
  - 将 Query/Key 拆分为正负部分并改写 Quest 公式，使主运算运行在 Tensor Core，以实现矩阵高吞吐计算。
  - 大幅提升关键簇选择速度（最长序列时最高提升 5×）。  

### 4. 总体算法流程（Algorithm 3）
1. 对 Query 归一化 → K-means 聚类获取 `Aq, Cq`；  
2. 对 Key 向量执行 MultiStageClustering 获取 `Ak, Ck, F`；  
3. 若该层难以压缩（F=True）则执行全注意力，否则：  
   - 调用 TensorQuest 计算关注度评分；  
   - 按 Top-k 选取关键簇；  
   - 仅在关键簇上执行注意力计算。
