## 摘要
自回归视频扩散作为一种用于流式视频合成的有前景范式正在兴起，其中步蒸馏（step distillation）是加速推理的主要手段。推测解码（speculative decoding）是大型语言模型中的主流加速策略，但其是否能有效适配自回归视频生成仍然是一个开放问题，因为视频块是连续的时空张量，没有基于 token 级分布的精确拒绝采样机制。我们提出 SDVG，它通过使用图像质量路由器将 token 验证替换为评估机制，从而将推测解码引入基于块的自回归视频扩散。一个 13 亿参数的 drafter 通过四步去噪生成候选视频块；每个视频块经 VAE 解码，并通过采用最差帧聚合（取每帧奖励的最小值以捕捉平均可能掩盖的单帧伪影）的 ImageReward 进行评分。评分超过固定阈值 τ 的视频块被接受进入 140 亿参数目标模型的 KV 缓存，其余视频块由目标模型重新生成。两个额外的设计决策被证明至关重要：第一块视频总是强制拒绝以锚定场景构成，且 τ 作为一个单一调节参数用于描绘平滑的质量-速度帕累托前沿（Pareto frontier）。在 MovieGenVideoBench 的 1003 个提示（832×480）上，SDVG 在 τ = -0.7 时以 1.59 倍加速保留了 98.1% 的仅目标模型 VisionReward 质量（0.0773 对比 0.0788），并在 95.7% 质量保留下实现了 2.09 倍速度，同时相对于仅 drafter 生成持续超过 +17% 的性能提升。该框架无需训练、无需架构修改，可无缝集成到现有的自回归视频生成流程中。

## Abstract
Autoregressive video diffusion is emerging as a promising paradigm for streaming video synthesis, with step distillation serving as the primary means of accelerating inference. Whether speculative decoding, the dominant acceleration strategy for large language models, can be effectively adapted to autoregressive video generation remains an open question, because video blocks are continuous spatiotemporal tensors with no token-level distribution for exact rejection sampling. We introduce SDVG, which brings speculative decoding to block-based autoregressive video diffusion by replacing token verification with an image-quality router. A 1.3B drafter proposes candidate blocks via four denoising steps; each block is VAE-decoded and scored by ImageReward using worst-frame aggregation--taking the minimum per-frame reward to catch single-frame artifacts that averaging would mask. Blocks scoring above a fixed threshold tau are accepted into the 14B target's KV cache; the rest are regenerated by the target. Two additional design choices prove critical: the first block is always force-rejected to anchor scene composition, and tau serves as a single knob that traces a smooth quality-speed Pareto frontier. On 1003 MovieGenVideoBench prompts (832x480), SDVG retains 98.1% of target-only VisionReward quality (0.0773 vs. 0.0788) at a 1.59x speedup with tau=-0.7, and reaches 2.09x at 95.7% quality retention--while consistently outperforming draft-only generation by over +17%. The framework is training-free, requires no architectural changes, and can be seamlessly integrated into existing autoregressive video generation pipelines.

## 2. 提出的方法论：SDVG 框架

### 核心思想
- 提出 **SDVG（Speculative Decoding for Video Generation）**：一种**无需训练、无需修改模型结构**的推测式视频生成方法。  
- SDVG 通过一个“草稿模型（drafter）”生成候选视频块，用“图像质量路由器（image-quality router）”决定是否接受该块；若评分过低，则由“大模型（target）”重新生成。

### 关键技术细节
1. **双模型协作结构**
   - **Drafter**：1.3B 参数的小模型，用较少的去噪步（4 步）提议候选视频块；
   - **Target**：14B 参数的大模型，只有在候选块不达标时重新生成。

2. **块级推断流程**
   - 每个视频被拆为多个块（block），每块包含若干帧；
   - 对于第 0 块（block 0），始终由 target 重生成，以锚定场景构图；
   - 对 block 1–8：
     - drafter 生成候选视频块；
     - 通过编码-解码管线（VAE 解码）生成帧；
     - 使用 **ImageReward** 模型打分；
     - 若最小帧得分 \( q_b = \min_i R(f_i, p) ≥ τ \)，接受该块并存入 target KV 缓存；
     - 否则拒绝并由 target 模型重生成。

3. **最差帧聚合（worst-frame aggregation）**
   - 取每个块内帧的最小评分，避免因平均值掩盖单帧伪影导致的视觉闪烁；
   - 实验证明这种策略优于平均帧打分。

4. **阈值控制 (\(τ\))**
   - 单一阈值参数 \(τ\) 控制质量与速度的平衡；
   - 不同 τ 值构成平滑的“质量–速度帕累托前沿”。

## 3. 实验设计与对比

- **数据集 / 测试基准**
  - 使用 **MovieGenVideoBench**（共 1003 条文本提示），涵盖风景、人物、动物、电影镜头等多类型场景；
  - 输出分辨率为 **832×480**。

- **性能指标**
  - **VisionReward (VR)：** 多维视频评估指标，衡量视觉质量、时序一致性、运动自然性和文本一致性；
  - **Time / Speedup：** 每个视频生成的平均推理时间及相对加速倍数；
  - **Accept Rate：** 被路由器判定为合格的块占比。

- **对比方法**
  1. **Target-only**：全部由大模型生成（最高质量，最慢速度）；
  2. **Draft-only**：全部由小模型生成（最快但质量最低）；
  3. **SDVG (ours)**：基于阈值 τ 的动态路由；
  4. **随机路由（Random routing）**与**平均帧打分**等消融设定。

- **主要结果**
  - 在 τ = −0.7 时：
    - VisionReward：0.0773（仅低于 target-only 的 0.0788）；
    - 加速比：**1.59×**；
    - 质量保持：**98.1%**；
    - 块接受率：73.1%；
  - 在 τ = −2.5 时：
    - 加速至 **2.09×**；
    - 质量保持仍达 **95.7%**；
  - 始终明显优于 draft-only（质量提升约 +17%）。

## 7. 方法优点与亮点

- **创新性**：首次将 speculative decoding 机制迁移至连续视频张量领域；
- **灵活性**：无需修改架构、无需额外训练、易于集成；
- **鲁棒性**：通过最差帧聚合有效检测局部伪影；
- **可控性**：单一阈值即可平衡质量与速度；
- **可组合性**：理论上可与其他加速策略（如 step distillation、trajectory stitching）叠加使用。

## 8. 不足与局限

- **分布偏差**：非精准拒绝采样机制导致 drafter 分布与目标分布间存在偏移；
- **路由信号局限**：ImageReward 属于静态图像奖励模型，无法捕捉时序一致性与运动质量；
- **计算浪费**：对被拒绝块，drafter 与解码操作都是无效计算；
- **适用范围**：仅验证于 Self-Forcing 风格的自回归扩散模型；
- **实验覆盖**：只在 MovieGenVideoBench 进行验证，存在数据集泛化性问题。
