## Abstract
Current text-to-image models struggle to provide precise camera control using natural language alone. In this work, we present a framework for precise camera control with global scene understanding in text-to-image generation by learning parametric camera tokens. We fine-tune image generation models for viewpoint-conditioned text-to-image generation on a curated dataset that combines 3D-rendered images for geometric supervision and photorealistic augmentations for appearance and background diversity. Qualitative and quantitative experiments demonstrate that our method achieves state-of-the-art accuracy while preserving image quality and prompt fidelity. Unlike prior methods that overfit to object-specific appearance correlations, our viewpoint tokens learn factorized geometric representations that transfer to unseen object categories. Our work shows that text-vision latent spaces can be endowed with explicit 3D camera structure, offering a pathway toward geometrically-aware prompts for text-to-image generation. ## 一、研究背景与核心问题

- **研究动机**  
  当前的文本到图像（Text-to-Image, T2I）模型在语义与视觉真实性方面已取得显著进展，但仅靠自然语言指令难以实现**精确的视角（camera viewpoint）控制**。例如，诸如“背面视图”“30°俯视角”这类指令往往生成不稳定或错误的姿态。  
  原因在于：
  - 自然语言在几何参数上不连续且模糊；
  - 训练数据偏向于常见前视角；
  - 现有控制方法（如 ControlNet、Compass Control）或依赖图像输入、或仅支持单一旋转轴。  

- **研究目标**  
  本文旨在构建一种能够在**纯文本条件下精确控制相机姿态**的框架，使生成图像在保持语义一致与视觉质量的同时具备真实的几何一致性与全局场景理解。

## 二、方法论：核心思想与技术架构

### 1. 核心思想

通过引入**可学习的参数化相机视点标记（Viewpoint Tokens）**，实现文本空间与几何空间的融合，使得生成模型能理解显式相机结构。

### 2. 相机参数化与编码流程

- **视点定义**  
  在对象中心坐标系内，固定对象正前方为 x 轴正方向。  
  相机视点参数采用五维向量：
  \[
  \theta = (\theta_{az}, \theta_{el}, r, \theta_{pitch}, \theta_{yaw})
  \]
  其中分别表示：
  - *Azimuth* 水平角；
  - *Elevation* 垂直角；
  - *r* 与对象直径归一的距离；
  - *Pitch* 下倾角；
  - *Yaw* 左右旋转角。  

- **标记编码**  
  - 参数首先经函数 \
    \(\phi(\theta) = [\sin(\theta_{az}), \cos(\theta_{az}), \theta_{el}, r, \theta_{pitch}, \theta_{yaw}]\)
    编码周期性信息；
  - 再输入三层带 ReLU 激活的 MLP，得到标记向量 \( e_{view} \)；
  - 将该视点标记嵌入文本标记序列，与文本一起进入扩散模型或多模态生成模型（如 Harmon）。

### 3. 模型训练策略

- 将相机标记与文本嵌入联合训练；
- 采用**两部分数据集：**
  1. **3D 渲染集** — 提供强几何监督；
  2. **逼真增强集** — 提供真实外观和多样背景；
- 联合微调既能学到几何一致性，又防止模型退化或丧失语义生成能力。

## 三、实验设计与比较设置

### 1. 使用的数据集

- **渲染数据集**  
  - 来源：TexVerse 3D 模型库（共 3,111 个对象，含动物、车辆、人物、家具等）；
  - 每个对象渲染约 120 个不同视角，生成 约 37 万张图像；
  - 背景透明，用于几何学习。

- **逼真增强数据集**
  - 从渲染集中选取 800 个高质量对象；
  - 使用 *Gemini 2.5 Flash Image (Nano Banana)* 生成包含真实背景和光照的图像；
  - 过滤后的有效图像约 6,600 张；
  - 提供真实感与多样性。

### 2. 评价基准（Benchmark）

包括两个方面：
- **几何准确度**：通过训练视角回归器计算生成图像的角度误差（azimuth、elevation、yaw、pitch、radius）。
- **语义一致性**：采用 CLIP 相似度与 GenEval 指标（颜色和单物体一致性）。

### 3. 对比方法

| 方法 | 输入类型 | 控制维度 |
|------|-----------|----------|
| ControlNet-Depth [50] | 图像 + 深度图 | 有完全几何监督（oracle baseline） |
| Stable-Virtual-Camera [52] | 图像 + camera | 基于单图像新视图合成 |
| Compass Control [27] | 文本 + 方位标记 | 仅支持水平旋转 |
| **本文方法** | 文本 + 相机标记 | 支持多维相机参数控制 |

## 八、不足与局限

- **视角覆盖有限**：训练集仅覆盖仰角 [0°, 45°]，缺少极端仰视/俯视、滚转角与镜头内部参数变化。

- **细部质量问题**：在人脸或复杂细节处偶有退化。
- **算力限制**：仅使用单卡训练，推测扩展至超大模型时仍需验证可扩展性。
